# Video Pipeline プレゼン スライド構成

---

## Part 1: イントロ（なぜこれを作ったか）

### スライド 1: タイトル
- **Video Pipeline — AI動画制作キット**
- 「撮影したMP4 → YouTube動画」を自動化する

### スライド 2: Before / After
- **Before（従来の動画編集）**
  - Premiere Pro / DaVinci Resolve を覚える必要がある
  - カット、字幕、SE、BGM… 全部手作業
  - 10分の動画に6〜10時間かかる
- **After（Video Pipeline）**
  - コマンドを15回打つだけ
  - AIが字幕のスタイルもSEのタイミングも自動判定
  - 編集ソフトの操作を一切覚えなくていい

### スライド 3: 全体像（1枚で伝える）
- **入力**: 撮影したMP4 + BGM
- **処理**: 15個のスラッシュコマンドを順番に実行
- **出力**: ジャンプカット・字幕・テロップ・スライド・SE・BGM付きのMP4
- 図: MP4ファイル → [Video Pipeline] → YouTube動画

## Part 2: 3フェーズの概要

### スライド 4: 3つのフェーズ
```
フェーズ1: 素材を整える（Step 01〜08）
    ↓
フェーズ2: 演出データを作る（Step 09〜11）
    ↓
フェーズ3: 組み立てて書き出す（Step 12〜15）
```
- たとえるなら：
  - フェーズ1 = 食材の下ごしらえ
  - フェーズ2 = レシピを決める
  - フェーズ3 = 調理して盛り付ける

### スライド 5: 技術スタック一覧
| 役割 | 技術 |
|------|------|
| 動画カット | FFmpeg |
| 文字起こし | Whisper (mlx-whisper) |
| スライド画像化 | Puppeteer |
| 動画フレームワーク | Remotion (React) |
| AI判断・自動化 | Claude Code |
| 最終書き出し | Remotion render |

## Part 3: フェーズ1 — 素材を整える（Step 01〜08）

### スライド 6: Step 01 — 動画コンテキスト
- **やること**: 「誰に」「何を伝える」動画かを決める
- **なぜ必要？**: この情報がないとAIが適切な演出を選べない
  - 例: テニス動画なら「ポーチ」は専門用語 → 強調テロップ
  - 例: 料理動画なら「ポーチ」は普通の単語 → 通常テロップ
- **出力**: `video-context.md`
- **ユーザーがやること**: ターゲットと趣旨を答えるだけ

### スライド 7: Step 02 — 素材チェック
- **やること**: 必要なファイルが揃っているか確認
- チェックリスト:
  - ✅ 撮影動画（MP4）→ `public/video/`
  - ✅ BGM（MP3）→ `public/bgm/`
  - ✅ SE（効果音）→ `public/se/` ← 自分で用意
- `ffprobe` で動画の解像度・FPS・長さを自動取得
- **ユーザーがやること**: ファイルを入れておくだけ

### スライド 8: Step 03 — ジャンプカット
- **やること**: 動画の無音部分を自動検出してカット
- **技術**: FFmpeg の `silencedetect` フィルタ
  - パラメータ: `noise=-30dB`, `d=0.3`（0.3秒以上の無音を検出）
- **Before → After のイメージ**:
  ```
  Before: 「えーと...[0.8秒無音]...次に...[1.2秒無音]...」
  After:  「えーと、次に、」（無音が消えてテンポUP）
  ```
- **出力**: `public/video/〇〇_cut.mp4`
- **ユーザーがやること**: 何もしない（自動）

### スライド 9: Step 04 — 文字起こし
- **やること**: AIが音声を聞いてテキスト+タイムスタンプに変換
- **技術**: Whisper（Apple Silicon向け高速版 mlx-whisper）
- **出力例**:
  ```json
  {
    "word": "テニス",
    "start": 3.42,
    "end": 3.88
  }
  ```
  → 「3.42秒目から3.88秒目に"テニス"と言った」
- 全単語にこのタイムスタンプがつく → 字幕の正確なタイミングの元データ
- **ユーザーがやること**: 何もしない（自動）

### スライド 10: Step 05 — 文字起こし修正
- **やること**: Whisperの聞き間違いを台本と照合して修正
- **なぜ必要？**: AIでも聞き間違える
  - 例: 「前衛」→「全営」
  - 例: 「ポーチ」→「ポーチー」
- 台本（ナレーション原稿）があれば照合して自動修正
- **ユーザーがやること**: 台本を貼り付ける（あれば）

### スライド 11: Step 06 — スライド生成
- **やること**: 台本の内容からHTMLスライドを自動生成
- 7種類のテンプレート:
  - title（タイトル）
  - three-cards（3枚カード）
  - two-columns（2列比較）
  - steps（ステップ形式）
  - big-message（大きいメッセージ）
  - closing（まとめ）
  - three-tactics（3つの戦術）
- デザイン: ライムイエロー `#CCFF00` + ダーク `#121212`
- **スライド不要な動画ならスキップOK**

### スライド 12: Step 07 — スライド画像化
- **やること**: HTMLスライドをPNG画像にキャプチャ
- **技術**: Puppeteer（ヘッドレスブラウザ）
  - ブラウザを見えない状態で起動 → HTMLを表示 → スクリーンショット
- **出力**: `public/slides/slide-01.png`, `slide-02.png`, ...
- サイズ: 1280×720px

### スライド 13: Step 08 — ブロック分割
- **やること**: スライドを「段階的に表示」するための分割画像を作成
- **なぜ必要？**: 3項目あるスライドを一気に見せるのではなく、1→2→3と順番に出したい
- **例（3枚カードのスライド）**:
  ```
  slide-04-title.png    ← タイトルだけ
  slide-04-col1.png     ← タイトル + 1枚目
  slide-04-col2.png     ← タイトル + 1枚目 + 2枚目
  slide-04-full.png     ← 全部表示
  ```
- テロップの出現に合わせてブロックが1つずつ出る → 視聴者の目が自然に誘導される

## Part 4: フェーズ2 — 演出データを作る（Step 09〜11）

### スライド 14: フェーズ2の概要
- このフェーズでは「どこに何を出すか」をデータとして定義する
- 3つの設定ファイルを作る:
  - `templateConfig.ts` → テロップの見た目の定義
  - `telopData.ts` → 全字幕のデータ
  - `slideTimeline.ts` → スライドの表示タイミング

### スライド 15: Step 09 — テンプレート設定
- **やること**: テロップ（字幕）の見た目を定義する
- 15種類以上のテンプレート:

| テンプレート | 見た目 | 使う場面 |
|------------|--------|---------|
| 通常 | 紺文字+白フチ | 普通の発言 |
| 通常+強調 | 紺+赤文字 | 1語だけ目立たせたい |
| 強調グラデ | 赤→金グラデ | 重要ワード |
| ネガティブ | 白文字+黒グロー | 警告・失敗 |
| 第三者発言 | 白+グレーフチ+「」 | 他人の言葉 |
| 代弁 | 吹き出し | 視聴者の気持ち |

- フォントも2種類使い分け:
  - 丸ゴシック（M PLUS Rounded 1c）→ 情報系
  - 明朝体（Shippori Mincho）→ 感情系

### スライド 16: Step 10 — テロップデータ作成（重要）
- **やること**: 全発話に字幕を割り当てる（カバー率90%以上）
- **AIが自動判定する内容**:
  - この発言は「強調」？「ネガティブ」？「通常」？
  - どのSEを鳴らす？（強調/ ポジティブ/ ネガティブ）
- **判定ロジックの例**:
  ```
  「500人以上を指導」→ 数字あり → EmphasisTelop2（金グラデ大）
  「それはダメです」→ ネガティブ → NegativeTelop（白+黒グロー）
  「ポーチが大事」→ 専門用語 → EmphasisTelop（赤グラデ）
  「こんにちは」→ 特になし → NormalTelop（通常字幕）
  ```
- **出力例**:
  ```typescript
  {
    text: "500人以上を指導",
    startFrame: 450,
    endFrame: 510,
    template: "emphasis2"
  }
  ```

### スライド 17: Step 10 の判定フロー図
```
発話テキスト
  ↓
数字・金額がある？ → YES → 強調大（金グラデ）+ SE:ポジティブ
  ↓ NO
ネガティブワード？ → YES → ネガティブ（白+黒）+ SE:ネガティブ
  ↓ NO
専門用語？ → YES → 強調（赤グラデ）+ SE:ポジティブ
  ↓ NO
問いかけ？ → YES → ネガティブ（共感演出）+ SE:ネガティブ
  ↓ NO
通常テロップ（SEなし）
```

### スライド 18: Step 11 — スライドタイムライン
- **やること**: どのスライドを何秒目〜何秒目に表示するか定義
- **出力例**:
  ```typescript
  {
    slideNumber: 4,
    startFrame: 900,   // 30秒目
    endFrame: 1200,    // 40秒目
    image: "slides/blocks/slide-04-col1.png",
    motion: "panRight"  // Ken Burnsモーション
  }
  ```
- ブロックスライドの場合は同じスライド番号で複数エントリ → 段階的に表示
- フェードインアニメーション（8フレーム）で自然に切り替わる

## Part 5: フェーズ3 — 組み立てて書き出す（Step 12〜15）

### スライド 19: フェーズ3の概要
- フェーズ1で整えた素材 + フェーズ2で作った演出データ
- → 1つのReactコンポーネントに統合 → MP4に書き出す

### スライド 20: Step 12 — メインコンポジション構築（核心）
- **やること**: 全要素を1つのReactコンポーネントに統合する
- 5つのレンダリングシステムが動画の各レイヤーを担当:

```
┌─────────────────────────────────┐
│  ⑤ SE + BGM（音声レイヤー）      │ ← 聞こえる
├─────────────────────────────────┤
│  ④ テロップ（字幕）    z:10〜15  │ ← 一番上
├─────────────────────────────────┤
│  ③ ワイプ（丸い小窓）   z:8      │ ← 話者の顔
├─────────────────────────────────┤
│  ② スライド背景        z:5      │ ← 解説スライド
├─────────────────────────────────┤
│  ① メイン動画          z:0      │ ← 一番下
└─────────────────────────────────┘
```

### スライド 21: Step 12 — レイヤー詳細
- **① メイン動画**: カット済みMP4を全画面表示
- **② スライド背景**: slideTimeline に従ってスライド画像を切り替え
  - Ken Burnsモーション（ゆっくりパン/ズーム）で動きをつける
  - ブロックスライドはフェードイン（8フレーム）で段階表示
- **③ ワイプ**: スライド表示中に話者の顔を丸い小窓で見せる
  - 右上280×280px、円形クリップ
- **④ テロップ**: telopData に従って字幕を表示
  - テンプレートに応じたスタイル・アニメーション適用
- **⑤ SE + BGM**: テロップ出現と同時にSEを鳴らす + BGMは全体にかける

### スライド 22: Step 12 — SEの自動選択ロジック
- テロップのテンプレートに応じたフォルダからランダム選択
  ```
  強調テロップ → se/強調/ の中からランダム
  ネガティブ  → se/ネガティブ/ の中からランダム
  通常テロップ → SEなし
  ```
- ただし:
  - 直近2回と同じSEにならない（単調さ防止）
  - startFrame をシードに使う（再生するたびに同じ結果）
  - 連続SEは最低20フレーム（0.7秒）空ける

### スライド 23: Step 13 — Remotion に登録
- **やること**: 作ったコンポジションをRemotionに登録
- `Root.tsx` に以下を記述:
  - 動画ID
  - 解像度: 1920×1080
  - FPS: 30
  - 総フレーム数（= 動画の長さ × 30）
- これでプレビュー・書き出しが可能になる

### スライド 24: Step 14 — プレビュー確認
- **やること**: 重要なフレームのスクリーンショットを撮って目視確認
- チェックポイント:
  - 字幕の位置・サイズは正しいか
  - スライドの切り替えタイミングは合っているか
  - ワイプの位置は適切か
  - 文字が見切れていないか
- 問題があればその場で修正

### スライド 25: Step 15 — 最終レンダリング
- **やること**: ReactコンポーネントをMP4動画に書き出す
- コマンド1つで完了:
  ```bash
  npx remotion render MainComposition public/output/MainComposition.mp4
  ```
- 書き出し後の自動チェック:
  - ファイルサイズ
  - 動画の長さ
  - 解像度（1920×1080）
- → **完成！YouTubeにアップロードできる状態**

## Part 6: Remotionとは？（技術補足）

### スライド 26: なぜ Remotion を選んだか
- **Remotion = Reactで動画を作るフレームワーク**
- メリット:
  - コードで動画を定義 → 再現性100%、バージョン管理できる
  - Reactの知識がそのまま使える
  - プログラムで制御 → AIとの相性が抜群
- 従来の動画編集ソフト（Premiere等）との違い:
  - GUI操作 → AIが操作できない
  - プロジェクトファイル → テキストベースではない
  - **Remotionはコード = AIが読み書きできる**

### スライド 27: Remotion の仕組み
```
React コンポーネント
  ↓
各フレーム（1/30秒ごと）をレンダリング
  ↓
フレーム画像を連結
  ↓
MP4動画
```
- 30fpsの10分動画 = 18,000フレーム
- 各フレームは「このフレーム番号のとき、何を表示するか」をReactで定義
- `useCurrentFrame()` で現在のフレーム番号を取得 → 条件分岐で表示制御

## Part 7: AIは何をしているのか

### スライド 28: AIの役割まとめ
- **AIが判断すること**:
  - Step 10: この発言は強調？ネガティブ？通常？（テロップスタイル判定）
  - Step 05: Whisperの聞き間違いを台本と照合して修正
  - Step 06: 台本の内容からスライドのテンプレートを選択
  - Step 12: 全要素を適切に統合するReactコードを生成
- **AIではなくルールで動くこと**:
  - Step 03: FFmpegが無音を検出してカット（ルール通り）
  - Step 04: Whisperが音声をテキストに変換（AI だが固定処理）
  - Step 07-08: Puppeteerがスクショを撮る（機械的処理）
  - Step 15: Remotionがフレームを連結してMP4に（機械的処理）

### スライド 29: CLAUDE.md — AIの行動ルールブック
- 全ルールが1つのファイルに書いてある（約500行）
- 主なルール:
  - テロップの色・フォント・サイズ → 全テンプレート固定値
  - 重複禁止 → 同じタイミングに字幕を2つ出さない
  - SEの選択方法 → フォルダ対応表 + 連続回避
  - 角丸禁止 → `borderRadius` は使わない
  - 90%カバー率 → 発話の9割以上に字幕をつける
- **AIが勝手にデザインを変えない。ルールブックに従う。**

## Part 8: まとめ・デモ

### スライド 30: 全体フロー（最終まとめ）
```
MP4 + BGM
  ↓ step01: ターゲット・趣旨を決める
  ↓ step02: 素材チェック
  ↓ step03: 無音カット          ← FFmpeg
  ↓ step04: 文字起こし          ← Whisper
  ↓ step05: 誤変換修正          ← AI + 台本
  ↓ step06: スライドHTML生成     ← AI + テンプレート
  ↓ step07: スライド画像化       ← Puppeteer
  ↓ step08: ブロック分割         ← Puppeteer
  ↓ step09: テロップ設定定義
  ↓ step10: 全字幕データ生成     ← AI自動判定
  ↓ step11: スライドタイムライン
  ↓ step12: React統合           ← AI + Remotion
  ↓ step13: 登録
  ↓ step14: プレビュー確認
  ↓ step15: MP4書き出し          ← Remotion render
  ↓
YouTube動画 完成！
```

### スライド 31: 実際にやってみよう

  1. `git clone` でテンプレートを取得
  2. 動画とBGMを入れる
  3. `/step01-context` から順番に実行
