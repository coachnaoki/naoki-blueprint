更新履歴

naoki-blueprint のバージョンアップ履歴（Semantic Versioning）

追加された機能・改善の一覧です。新しい動画を作るときは bash 新規作成.sh が自動で最新版を取得します。手動で更新したい場合は bash アップデート.sh を実行してください。

v2.0.6 2026-04-28 v2.0 系
🎬 横動画版も「再起動なしで AUTO_RUN 直結」に対応

v2.0.5 でショート動画版だけ B案対応にしましたが、横動画版 (全20ステップ) も同じ体験にできるよう拡張しました。 これで横動画・ショート動画どちらでも、新規プロジェクト作成後に Claude Code セッションを切らずに step01 からそのまま動画制作に進めます。

  • 横動画版を B案対応に全面改修: .template/.claude/commands/new-video.md を PROJECT_DIR ベースの設計に変更、全20ステップを Read+Bash で順次実行
  • ROOT 版は薄いブートストラップに: 動画タイプによって step 構成 (20/14) が違うため、 ROOT 版は「新規作成」と「AUTO_RUN 起動」だけ担当し、AUTO_RUN モードではプロジェクト固有の new-video.md に委譲する設計に変更
  • 動画タイプの自動判定: step20-highlight-final/ があれば横動画、 step14-final/ があればショート動画、と .claude/skills/ の中身で自動判定

※ 次回 bash アップデート.sh で自動反映されます。

v2.0.5 2026-04-28 v2.0 系
🚀 /new-video を「再起動なしで AUTO_RUN 直結」できるように改善 (ショート動画版)

v2.0.4 でルートからでも /new-video が叩けるようになりましたが、新規プロジェクト作成後はまだ「新タブで cd projects/XXX → claude 起動」の3ステップが必要でした。 v2.0.5 ではこの再起動が不要になり、 同じ Claude Code セッションでそのまま step01 から AUTO_RUN を始められるようになりました。

  • 新規作成完了 → AUTO_RUN 直結: 「このセッションで AUTO_RUN を始める？」を選ぶだけで、Cursor 新ウィンドウ・claude 再起動なしで動画制作フローへ進める
  • 新規作成.sh のダウングレード事故を修正: version_track 対応にしたので、v2 ライセンスの方は origin/v2.0、v1/未指定の方は origin/main を見るようになり、ブランチ事故が起きなくなった
  • 各 step の SKILL.md を Read+Bash で実行: ROOT cwd / project cwd どちらでも動くよう、 cd "$PROJECT_DIR" && ... 形式で統一

※ 次回 bash アップデート.sh で自動反映されます。横動画版 (.template/) は次回リリースで対応予定です。

v2.0.3 2026-04-28 v2.0 系
✨ 将来 v3 リリース時の自動追従に対応

スプレッドシート S列 (version_track) が 空白 の方は、自動で最新メジャー版が降りてくる仕組みに変更しました。v1/v2 記載の方は従来通りそのバージョンに固定されます。

  • 空白の方 = 自動で最新版 (現状 v2.0、将来 v3 が出たら自動で v3.0 取得開始)
  • v1 記載の方 = main (v1 系列) に固定
  • v2 記載の方 = v2.0 ブランチに固定

※ あなたが何かする必要はありません。配布側で管理されます。
