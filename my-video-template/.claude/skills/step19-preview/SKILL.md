# Step 19: 最終プレビュー確認

BGM・画像・動画・スライドをすべて入れた最終状態を確認する。

## 前提条件
- 素材挿入（step11〜14）が完了していること
- スライドを入れる場合はstep15〜18も完了していること

## やること

### 1. 確認ポイントの選定

以下のフレームでスクリーンショットを撮影する：

- **冒頭（フレーム0付近）**: 最初の表示が正しいか
- **BGM区間**: 音声が正しく再生されるか（Studio再生で確認）
- **画像挿入区間**: 画像のz-index・タイミングが正しいか
- **動画クリップ区間**: Sequenceで正しく再生されるか
- **スライド表示中**: スライド背景・ワイプが正しいか
- **ブロック切り替え**: 段階表示のタイミングが正しいか
- **テロップ表示中**: テロップが画像・スライドの上に表示されているか
- **終盤**: 最後の表示が正しいか

### 2. スクリーンショット撮影

```bash
npx remotion still MainComposition --frame={フレーム番号} debug-final-{番号}.png
```

### 3. 確認

撮影したPNGを Read ツールで確認する。

チェック項目：
- [ ] スライド背景が正しく表示されている
- [ ] ブロック段階表示のタイミングが正しい
- [ ] テロップのスタイル・位置が正しい
- [ ] 円形ワイプが正しい位置・サイズで表示されている
- [ ] z-index が正しい（テロップが最前面）
- [ ] 画像挿入区間の表示が正しい
- [ ] 動画クリップが正しく再生される
- [ ] グリーンバック背景が正しく置換されている（該当する場合）
- [ ] 見出しバナーの表示タイミングが正しい（該当する場合）

### 4. スクショの即時削除（必須）

```bash
rm debug-final-*.png
```

### 5. ブラウザでプレビュー再生

Remotion Studioで通し再生して最終確認する。

### 6. 問題があれば修正

問題を発見した場合は該当ファイルを修正し、再度スクショで確認する。

## 完了条件
- 全要素（テロップ・スライド・画像・動画・BGM）の見た目を確認済み
- デバッグ用スクリーンショットをすべて削除済み
- 目立つ問題がないこと
