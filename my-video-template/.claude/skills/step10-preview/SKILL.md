# Step 10: プレビュー確認

`npx remotion still` でスクリーンショットを撮影し、視覚的に動画の各ポイントを確認する。

## 前提条件
- Step 09（コンポジション登録）が完了していること
- TypeScript ビルドが通ること

## やること

### 1. 確認ポイントの選定

以下の重要フレームでスクリーンショットを撮影する：

- **冒頭（フレーム0付近）**: 最初のスライドが正しく表示されるか
- **テロップ表示中**: テロップのスタイルが正しいか
- **スライド切り替え直後**: スライド遷移が正しいか
- **画像挿入区間**: 画像とワイプの切り替えが正しいか
- **終盤**: 最後のスライドが正しく表示されるか

引数 `$ARGUMENTS` でフレーム番号が指定されていればそのフレームを優先する。

### 2. スクリーンショット撮影

```bash
npx remotion still MainComposition --frame={フレーム番号} debug-{番号}.png
```

### 3. 確認

撮影したPNGを Read ツールで確認する。

チェック項目：
- [ ] スライド背景が正しく表示されている
- [ ] テロップのスタイル・位置が正しい
- [ ] 円形ワイプが正しい位置・サイズで表示されている
- [ ] z-index が正しい（テロップが最前面）
- [ ] 画像挿入区間ではワイプが非表示になっている

### 4. スクショの即時削除（必須）

**確認が終わったらすぐに削除する。**

```bash
rm debug-*.png
```

⚠️ デバッグ用スクショは確認→即削除。publicフォルダにゴミを残さない。

### 5. ブラウザでプレビュー再生

スクショ確認で問題がなければ、Remotion Studioをブラウザで開いてユーザーに確認してもらう。

```bash
npx remotion studio
```

ブラウザが自動で開かない場合は `open http://localhost:3000` で開く。

### 6. 問題があれば修正

問題を発見した場合は該当ファイルを修正し、再度スクショで確認する。

## 完了条件
- 主要フレームの見た目を確認済み
- デバッグ用スクリーンショットをすべて削除済み
- 目立つ問題がないこと
